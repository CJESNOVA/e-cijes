🚀 API de Diagnostic Complet

Récupérez toutes les données de diagnostic pour un utilisateur

� Authentification

📝 Description

L'API est maintenant sécurisée avec des tokens Bearer. Vous devez d'abord vous authentifier pour obtenir un token, puis l'inclure dans vos appels API.

🔑 Obtenir un token

POST /api/v1/auth/login
Corps de la requête (JSON) : 
{
  "email": "votre@email.com",
  "password": "votre_mot_de_passe"
}

✅ Réponse de connexion

{
  "success": true,
  "message": "Authentification réussie",
  "data": {
    "user": {
      "id": 123,
      "email": "votre@email.com",
      "name": "Nom Utilisateur"
    },
    "token": "1|abc123def456...",
    "token_type": "Bearer"
  }
}

🔒 Utiliser le token

Ajoutez le token dans l'en-tête Authorization :

Authorization: Bearer 1|abc123def456...

� Endpoint Principal

GET /api/v1/diagnostic/complet/{email} 🔐 Protégé

📝 Description

Cette API permet de récupérer toutes les données de diagnostic pour un utilisateur donné via son email, incluant :

  • Informations du membre associé à l'email
  • Uniquement les diagnostics avec diagnostictype_id = 1 (diagnostics personnels)
  • Modules, questions, réponses et scores détaillés
  • Statistiques calculées automatiquement par module et globales
  • Plans d'accompagnement associés au diagnostic

🔧 Paramètres

email (string, obligatoire) : Email de l'utilisateur dont on veut récupérer les diagnostics

🔗 Structure des Données

👤 Membre

  • Informations de base du membre
  • Diagnostics personnels (type 1)
  • Coordonnées complètes

📋 Diagnostic

  • Score global et commentaire
  • Type, statut et profil
  • Modules avec questions/réponses
  • Statistiques détaillées
  • Plans d'accompagnement

� Module

  • Informations de catégorie
  • Liste des questions
  • Statistiques par module
  • Position et description

� Question

  • Titre et position
  • Type et catégorie
  • Réponse choisie avec score
  • Réponse textuelle optionnelle

  const response = await fetch('http://localhost/cijet/public/api/v1/auth/login', {

⚛️ En React (version custom hook)

import { useState, useEffect } from 'react';

// Custom hook pour l'API de diagnostic
const useDiagnosticAPI = (userEmail) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const fetchDiagnostic = async (email) => {
    if (!email) return;
    
    try {
      setLoading(true);
      setError(null);
      
      const response = await fetch(
        `http://localhost/cijet/public/api/v1/diagnostic/complet/${encodeURIComponent(email)}`,
        {
          method: 'GET',
          headers: {
            'Accept': 'application/json',
            'Content-Type': 'application/json'
          }
        }
      );

      const result = await response.json();
      
      if (result.success) {
        setData(result.data);
      } else {
        setError(result.message);
      }
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchDiagnostic(userEmail);
  }, [userEmail]);

  return { data, loading, error, refetch: () => fetchDiagnostic(userEmail) };
};

// Utilisation du hook
const DiagnosticComponent = () => {
  const [email, setEmail] = useState('user@example.com');

🐍 En Python (requests)

import requests

# 1. Obtenir un token d'authentification
login_url = "http://localhost/cijet/public/api/v1/auth/login"
login_data = {
    "email": "votre@email.com",
    "password": "votre_mot_de_passe"
}

try:
    # Étape 1: Connexion
    login_response = requests.post(login_url, json=login_data)
    login_result = login_response.json()
    
    if not login_result['success']:
        print("Erreur de connexion:", login_result['message'])
        exit()
    
    token = login_result['data']['token']
    headers = {
        "Accept": "application/json",
        "Content-Type": "application/json",
        "Authorization": f"Bearer {token}"
    }
    
    # Étape 2: Récupérer les données de diagnostic
    diagnostic_url = "http://localhost/cijet/public/api/v1/diagnostic/complet/user@example.com"
    response = requests.get(diagnostic_url, headers=headers)
    data = response.json()
    
    if data['success']:
        print("Données de diagnostic:", data['data'])
        print(f"Membre: {data['data']['membre']['nom_complet']}")
        print(f"Nombre de diagnostics: {len(data['data']['membre']['diagnostics'])}")
    else:
        print("Erreur:", data['message'])
        
except requests.exceptions.RequestException as e:
    print("Erreur réseau:", e)

⚠️ Notes Importantes

🔍 Filtrage des diagnostics

L'API retourne uniquement les diagnostics avec diagnostictype_id = 1 (diagnostics personnels du membre).

📊 Calcul des statistiques

Les statistiques sont calculées automatiquement :
- score_total: Cumul des scores des réponses choisies
- score_maximum: Cumul des scores maximums par question
- pourcentage: (score_total * 100) / score_maximum

📋 Plans d'accompagnement

Les plans sont récupérés depuis l'accompagnement lié au diagnostic. Seuls les plans actifs (etat = 1) sont retournés.

🚀 Performance

L'API récupère toutes les données en une seule fois, ce qui peut être volumineux pour des utilisateurs avec beaucoup de diagnostics.

📄 Modules filtrés

Seuls les modules de type 1 sont inclus dans les résultats.

💾 Cache

Considérez l'ajout de cache pour les données qui ne changent pas fréquemment.

🔗 Relations Utilisées

  • User (recherche par email) → Membre (via user_id)
  • Membre → Diagnostic (via membre_id, filtré par diagnostictype_id = 1)
  • Diagnostic → Diagnosticresultat (via diagnostic_id)
  • Diagnosticresultat → Diagnosticquestion (via diagnosticquestion_id)
  • Diagnosticresultat → Diagnosticreponse (via diagnosticreponse_id)
  • Diagnosticquestion → Diagnosticmodule (via diagnosticmodule_id)
  • Diagnosticmodule → Diagnosticmodulecategory (via diagnosticmodulecategory_id)
  • Diagnostic → Accompagnement (via entreprise_id)
  • Accompagnement → Plan (via accompagnement_id)
