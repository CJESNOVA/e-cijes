API Structure des Diagnostics

Documentation pour l'API de récupération de la structure complète des diagnostics (modules, questions, réponses)

API Publique : Cette API ne nécessite pas d'authentification. Toutes les routes sont accessibles librement.

Structure Complète

GET /api/v1/diagnostic-structure/complete

Récupère la structure complète de tous les modules organisés par catégories, avec leurs questions et réponses possibles.

Filtrage : Uniquement les modules actifs (etat = 1) de type 1, organisés par ordre de position.

curl -X GET "http://localhost:8000/api/v1/diagnostic-structure/complete" \

Structure par Module

GET /api/v1/diagnostic-structure/module/{moduleId}

Récupère la structure détaillée d'un module spécifique avec ses questions et réponses possibles.

Filtrage : Uniquement les modules actifs (etat = 1) de type 1.

Exemple de requête :

Questions par Module

GET /api/v1/diagnostic-structure/module/{moduleId}/questions

Récupère uniquement les questions d'un module spécifique (sans les réponses possibles).

Filtrage : Uniquement les questions actives (etat = 1) des modules de type 1.

Exemple de requête :
curl -X GET "http://localhost:8000/api/v1/diagnostic-structure/module/1/questions" \
     -H "Content-Type: application/json"
Exemple de réponse :
{
    "success": true,
    "message": "Questions du module récupérées avec succès",
    "data": {
        "questions": [
            {
                "id": 1,
                "titre": "Comment gérez-vous le stress ?",
                "description": "Évaluez votre capacité à gérer le stress",
                "ordre": 1
            },
            {
                "id": 2,
                "titre": "Quelle est votre motivation ?",
                "description": "Évaluez vos motivations personnelles",
                "ordre": 2
            }
        ],
        "total": 8
    }
}

Structure par Profil d'Entreprise

Récupère la structure des diagnostics spécifique à un profil d'entreprise, organisée par catégories.

Filtrage : Modules génériques (entrepriseprofil_id = null) ou spécifiques au profil, uniquement les modules actifs de type 1.
